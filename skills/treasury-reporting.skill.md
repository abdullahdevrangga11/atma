# Skill: Treasury Reporting

> Read by `ReporterAgent` at runtime. Defines P&L methodology, baselines, and compliance-ready CSV export.

## Inputs

- `vaultHistory` — array of vault events (deposit, allocation, rebalance, withdraw, defensiveExit)
- `currentNAV` — current vault net asset value
- `entryNAV` — vault NAV at first deposit
- `period` — reporting period (`day`, `week`, `month`, `inception`)

## Outputs

```typescript
type WeeklyReport = {
  period: { start: Date, end: Date };
  actualNAV: bigint;        // current vault NAV in USDC
  actualPnL: bigint;        // actualNAV - entryNAV
  actualAPY: number;        // annualized
  baselines: {
    doNothing: { pnl: bigint, apy: number };
    usdcAaveOnly: { pnl: bigint, apy: number };
    usdyOnly: { pnl: bigint, apy: number };
  };
  outperformanceBps: {
    vsDoNothing: number;
    vsUsdcAaveOnly: number;
    vsUsdyOnly: number;
  };
  attestations: ERC8004Event[];  // all reputation events in period
  csvUrl: string;           // signed URL to exportable CSV
};
```

## P&L Computation

### Actual P&L

```
actualPnL = currentNAV - sum(deposits) + sum(withdrawals)
```

Account for time-weighted entries:
```
TWP = Σ (segmentNAV_change × time_in_segment / total_time)
actualAPY = TWP × (365 days / period_days)
```

### "Do Nothing" baseline

The user's deposited USDC, sitting idle in a non-yielding wallet.

```
doNothingPnL = 0  // USDC earns 0% APY in a wallet
doNothingAPY = 0
```

### "USDC Aave Only" baseline

The user's deposited USDC, supplied to Aave V3 Mantle USDC pool at live supply APY.

```
For each deposit at time t:
  positionValue(now) = deposit × (1 + avgAaveSupplyAPY)^(now - t)
usdcAaveOnlyPnL = Σ positionValues - Σ deposits
```

### "USDY Only" baseline

The user's deposited USDC, swapped to USDY at deposit time, held.

```
For each deposit at time t:
  usdyAmount = deposit × usdcUsdyExchangeRate(t)
  positionValue(now) = usdyAmount × usdyPrice(now)
usdyOnlyPnL = Σ positionValues - Σ deposits
```

### Outperformance

```
outperformanceVsX = (actualAPY - baselineX_APY) × 10000  // basis points
```

## Reporting Cadence

| Period | Trigger | Stored |
|---|---|---|
| Daily snapshot | midnight UTC | on-chain hash + off-chain JSON |
| Weekly report | Sunday 00:00 UTC | full report + CSV |
| Monthly report | 1st of month 00:00 UTC | full report + CSV |
| Defensive exit | on trigger | immediate report with reasoning |
| User-requested | on-demand | immediate |

## Reasoning Output

Each report includes a plain-English reasoning paragraph generated by the agent:

```
"This week ATMA outperformed 'do nothing' by 462 bps annualized (your $10K
earned $8.85 vs $0 sitting in USDC). Versus 'USDC Aave only' baseline,
ATMA outperformed by 47 bps thanks to mUSD's 0.18% boost via rebasing.
No defensive exits triggered. Risk Agent emitted 6 'warn' heartbeats on
Aave oracle deviation (Tuesday 2pm UTC, ~85 minute duration) but reverted
to 'ok' without action. All 3 agent identities active. Next snapshot:
Sunday 00:00 UTC."
```

## ERC-8004 Attestation Feed

For each report period, include all reputation events from Allocator, Risk, Reporter agents in chronological order with:
- Event type (`AllocationExecuted`, `RebalanceTriggered`, `RiskWarn`, `RiskTrigger`, `Heartbeat`, `Snapshot`)
- Block number
- Tx hash
- Reasoning hash (linked to off-chain JSON)
- Inline reasoning string

## CSV Export

The CSV is generated server-side, uploaded to Supabase Storage, and a signed URL valid for 7 days is returned.

For privacy-sensitive treasuries, the export can be encrypted with a user-provided passphrase before upload.

## Anti-patterns

- ❌ Don't claim outperformance if vault is in `RiskTriggered` or `DefensiveExit` state — clearly mark "defensive period"
- ❌ Don't compare to baselines that don't fit user policy (e.g., aggressive baseline if user is conservative)
- ❌ Don't bury attestations — surface them prominently for auditability
- ❌ Don't use round numbers in reasoning — exact basis points only
- ❌ Don't include unrealized P&L without flagging it as such
